Digital Derby

by: Brandon Shen

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make this game because I looked at the previous game which was an old arcade game and decided to make another arcade game that I had played earlier. I chose to make this arcade game because it seemed like a simple game that would be easy to make, but it turned out harder when I had to organize the timers.

2. What I Learned

I learnt that the users were getting bored of just dodging the cars. There was no time limit, therefore by just staying at the slowest speed possible and dodging the cars you could achieve a very high score. What I did to make it harder was to add a time limit so that to get a high score you would need to consistently go at high speeds before the time ran out.

3. Architecture of My Game

The two different objects that interact in the game are the racecar and the rival racecar. You control the racecar and enable it to move side to side to get past the rival racecars. What the rival racecars do is have varying speeds to force you to fit into tight spaces. If you hit the rival racecar you will lose the game. The rival racecar and the racecar both have their own class. The rival racecar is spawned in the ObjectManager class, and so is the normal racecar. Their properties are stored in their own respective classes. The ObjectManager class has to be called on to spawn the objects, which happens in the GamePanel class. The GamePanel class sets up the different stages in the game. Finally, we have the DigitalDerby class, which is the driver class of this game. It calls upon GamePanel to start the game and sets up the height and width of the game and is the JFrame that holds everything.

4. Technical Implementation

I think that one of the hardest things to do in this project is setting up the timers and being able to use them to add to a score. I also think that the spawn times for enemies was also interesting, as I had to make the times longer as the speed got higher or else it would be near impossible to pass by the enemies.

5. Most Challenging Part

The most challenging part of this project was working with a lot of classes and having to be able to figure out how to call on other classes in the right place. It was also challenging when I wanted to edit the size of the rival racecar I had to change the size in the GamePanel and also in ObjectManager. One difficulty I had to overcome is adding on a new game element when you've written everything else in the game. The difficulty with this was that I had to go back to look at the LeagueInvaders instructions and go from the start to make sure this new added game element worked with all the other elements and did what it was supposed to at the right time.

6. Future Fixes and Updates

If I were to continue working on this game I would have made the width smaller and would have written down all the things I want to put in the game at the start rather than just add on stuff I thought of on the way through. I also would've added other things that would be on the road.
